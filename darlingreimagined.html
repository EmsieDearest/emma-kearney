The Art of Accessibility

SCOPE

I've been a loyal reader of Darling Magazine since discovering its first issue while in the checkout line at Whole Foods. Their philosophy was written so eloquently on the back cover. It reminded me of the values my feminist mother instilled in me: that what I do means so much more than how I look doing it. I can be a fierce go-getter in my career and invest in my future. I can fight for the rights of others and still be a feminine woman. I have mad respect for the magazine and the thoughtfully designed spreads. I think their UI is fantastic and wouldn't change a thing.

PROBLEM

That being said, the development lacked basic accessibility criteria. It's obvious that it wasn't even conceived as part of the design and development process. This is unfortunately the norm. People with certain disabilities depend on assistive technology (or AT) to use the web. Such technologies as screen readers which relay text and interactive objects into speech. Braille machines do the same but automate into braille. AT run into all sorts of problems when the website or app isn't designed and developed with keyboard navigation the same way as with a mouse or touchscreen. Think of keyboard navigation as a ramp for a wheelchair. Only this ramp prevents access for people with visual, mobile and cognitive disabilities from using a product.

I am a certified web accessibility specialist and advocate for people with disabilities. After running a test on Darling's website, I discovered over 300 failures with their accessibility. Here is what I would do to fix a hypothetical article by Darling. These changes would meet universal compliance (or WCAG AA).



SOLUTION

Let's refer to the keyboard navigation as using the tab key. When tabbing through a page, the user needs to see which element they are currently focused on. This is helpful for people who are visually impaired or colorblind. You see this sometimes when clicking on a button and you see an outline. While it is standard practice to put an outline around an object on focus, it does put a damper on the integrity of the design. Instead, there are tricks where the focus outline appears only on keyboard events. One of the most important implementations is making sure all interactive objects have a tab index of 0. Using proper syntax (we'll get to this later) has some inherited functionality for tabbing and there is no need to add a tabindex to syntax tags. However, lots of use of anonymous tags such as divs and spans, you will need to add the proper tab index code.

Next, the navigation needs to have the option to skip to the main section. Sighted users can scan quickly to parts of the page they are most interested. Now imagine you are blind and forced to go through every single element on the page, including the nav categories and subcategories, the social media links, sidebars, etc. before you get to the main section. What if you accidentally hit the wrong key and had to start over from the page url. Talk about losing your patience! This "skip to main" button only becomes visible when the tab key has focused on the button. Use the tab key and try it on my site. Try it on The New York Times. Try it on your favorite site. Do they have one? If so, isn't this fun!?



Another problem often faced is when the user gets stuck on a sidebar or popup that won't close. Sighted users can click a close button to leave. When focus hasn't been programmed into these areas, either the user can't enter into them or the user can't exit out, causing the user to have to refresh the page and their lose their spot. This is why it's so important to include keyboard event handlers along with click or tap events. It's another layer of code but it's totally worth it.



Using specific syntax tags are superior to vague divs and spans. AT uses these tags for translation. For example, it is considered poor practice to cloak a button inside of a div and then program it to function like a button. Designers often do this to have greater design control. Webflow code generators are notorious for this. AT will not recognize the button.
